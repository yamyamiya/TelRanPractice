1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout [file-name]

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged [file name]

3.Как отменить ошибочный commit ?
git reset HEAD ~ [number]

4.Как создать новую ветку и переключится в нее?
git checkout -b [branch name]

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
В случае fast-forward merge текущая ветка не имеет дополнительных коммитов 
по сравнению с веткой, которую мы мержим, коммиты объединяются. 
В случае no fast-forward гит сохраняет полную историю коммитов в девелоп-бранче. 
Если есть изменения в текущей ветке, которых нет в ветке, которую мы хотим объединить, 
git выполнит объединение no fast-forward merge. При слиянии no fast-forward Git создает новый коммит мержа в активную ветку. 
Родительский коммит указывает на активную ветку и ветку, которую мы хотим объединить.

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
Когда две ветки, которые мы пытаемся смержить, имеют изменения в одной строке в одном и том же файле,
или несовпадающие изменения, или если одна ветка удалила файл, который изменила другая ветка, и так далее.
В этом случае Git попросит вас помочь решить, какой из двух вариантов мы хотим сохранить.
При попытке объединить ветки, Git покажет, где происходит конфликт. 
Мы можем вручную удалить изменения, которые не хотим сохранять, сохранить изменения, снова добавить файл и закоммитить изменения.

7.Можно ли удалить ветку? Как? 
git branch -d [branch name]
